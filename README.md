# restaurantQueuingSimulator
A Java simulation system modelling restaurant table allocation under different queuing strategies.


# How to run
## Compile
```
find src -name "*.java" -print | xargs javac -d out/
```

## Run simulation:
Text-based. Runs every combination of algorithm, restaurant setting, and customer scenario and prints statistics.
```
java -cp out com.restaurant.Main data/restaurant_settings.json data/customer_scenarios.json
```

## Run UI version:
Swing GUI. Select a restaurant, scenario, and algorithm, then step through each event by clicking next. The floor plan shows each table's occupancy, current group, and seat fill.
```
java -cp out com.restaurant.InteractiveUI data/restaurant_settings.json data/customer_scenarios.json
```

## Run Generator:
Enter one or more sample mean arrival rates (groups/hour) and the program generates realistic customer arrival scenarios using Poisson inter-arrival times, saving them to `data/custom_scenarios.json`. Also supports adding and deleting scenarios.
```
java -cp out com.restaurant.PoissonGenerator
```

## Data Files

### `restaurant_settings.json`
Each entry defines a restaurant floor:
- `id` / `name` - unique identifier and display name
- `time_limit` - maximum dining minutes per table; `0` means no limit
- `tables` - array of `{tableId, capacity}` objects
- `type` - type of the restaurant
- `description` - description of the restaurant for more information about the setting

### `customer_scenarios.json`
Each entry defines a customer arrival flow:
- `id` / `name` - unique identifier and display name
- `arrivals` - array of groups, each with:
  - `groupId`, `groupSize`, `preferredTableSize`
  - `isVip` - boolean VIP flag
  - `arrivalTime` - minutes from simulation start
  - `diningDuration` - intended dining minutes (may be capped by `time_limit`)

### `custom_scenarios.json`
Auto-generated by `PoissonGenerator`. Same format as `customer_scenarios.json`. Loaded automatically by both Main and InteractiveUI if the file exists.


## Program File Description

### `Main.java`
Program 1: batch runner - fins statistics of the stimulation for all algorithms * restaurants * scenarios combinations
### `InteractiveUI.java`
Program 2: Swing GUI with step-by-step floor visualisation for each combination
### `PoissonGenerator.java`
Program 3: Interactive Poisson arrival scenario generator to create custom customer arrival settings

### `data/restaurant_settings.json `
Restaurant settings, with table capacities and time limits
### `data/customer_scenarios.json`
Hand-crafted customer arrival flows
### `data/custom_scenarios.json`
Auto-generated Poisson arrival scenarios (created by PoissonGenerator)

### `src/com/restaurant/model/Table.java`
Table entity: capacity, occupancy, time-limit enforcement
### `src/com/restaurant/model/CustomerGroup.java`
Customer group entity: size, VIP flag, arrival/dining times, wait tracking
### `src/com/restaurant/model/RestaurantSetting.java`
Holds a list of Tables and the dining time_limit for a restaurant
### `src/com/restaurant/model/CustomerScenario.java`
Holds a named list of CustomerGroup arrivals

### `src/com/restaurant/algorithm/QueueAlgorithm.java`
Interface all algorithms implement
### `src/com/restaurant/algorithm/SingleQueueStaffGuided.java`
One FIFO queue; staff assigns smallest fitting table
### `src/com/restaurant/algorithm/SingleQueueSelfService.java`
One FIFO queue; customers pick their preferred table size
### `src/com/restaurant/algorithm/GroupSizeBasedQueue.java`
Three size-band queues (1-2, 3-4, 5+); larger bands prioritised when arrival times are tied
### `src/com/restaurant/algorithm/PriorityQueueAlgorithm.java`
VIP groups jump ahead; within same priority, FCFS

### `src/com/restaurant/simulation/SimulationStep.java`
Snapshot of one simulation event (tables, queue, event description)
### `src/com/restaurant/simulation/SimulationEngine.java`
Drives any algorithm; computes all output statistics
### `src/com/restaurant/simulation/SimulationResult.java`
All output metrics: utilisation, wait, fairness, service levels...

### `src/com/restaurant/util/JsonLoader.java`
JSON parser
### `src/com/restaurant/util/CsvExporter.java`
Writes all SimulationResult objects to a CSV file
